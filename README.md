# 🎮 게임 스타터 (빈 뼈대)

**아직 게임이 없어요.** `prd.md`에 내 게임을 기획하고, AI(OpenCode)와 함께 만들어 나가는 **시작점**이에요.

> 🎯 **목표:** 설치 없이 **브라우저에서 도는 웹 게임**을 `index.html` 하나로 만들어 → **GitHub Pages로 배포**(친구와 링크 공유)!

## 폴더에 든 것
| 파일 | 설명 |
|---|---|
| `AGENTS.md` | **AI(OpenCode)가 자동으로 읽는 프로젝트 안내·규칙** |
| `prd.md` | **게임 기획서** — 뼈대 + 채운 예시. 여기부터 채워요 |
| `design.md` | 🎨 색·폰트·게임 느낌 가이드 (AI가 읽음) |
| `context.md` | 프로젝트 기억 (다음에 할 일 메모) |
| `index.html` | **빈 뼈대** — 여기에 게임이 만들어져요 |
| `assets/` | 그림(SVG)·소리 에셋 · `RESOURCES.md`(무료 출처) |

## 만드는 순서
1. **기획** — `prd.md`를 내 게임으로 채워요. (최소한만 채워도 OK · 미리보기 `Ctrl+Shift+V`)
2. **AI와 다듬기** — OpenCode에 검토를 받아요:
   ```
   이 폴더의 prd.md를 검토해줘. 빠진 항목이나 더 구체화할 부분,
   넣으면 좋을 요소를 나에게 하나씩 질문해줘.
   ```
   질문에 답하며 채우면 기획이 탄탄해져요.
3. **만들기** — 어느 정도 됐으면:
   > 💡 **계획부터** — 바로 만들지 말고 *"개발 계획(plan.md)을 먼저 세워줘"* → 검토 → *"1단계만 실행"* 순으로 시키면 정확해요. **고칠 때도 똑같이.** (수업 자료 4장)
   ```
   이 폴더의 prd.md를 참고해서 index.html에 웹게임을 만들어줘.
   assets 폴더의 이미지·소리를 쓰고, 초보용 한국어 주석을 달아줘.
   ```
   저장 → Live Server로 확인 → **하나씩** 고쳐 부탁 (이동 → 떨어지는 것 → 점수 → 게임오버).
   > 💡 **(선택) 목업 먼저** — 흐름이 헷갈리면 *"눌러지는 목업으로 화면 흐름만 먼저 보여줘"* 로 확인한 뒤 게임을 완성해도 좋아요. (수업 자료 4장)
4. **이어가기** — 멈추기 전 `context.md`의 "다음에 할 일"을 갱신.

## ✅ 올리기 전 체크리스트
- [ ] 파일 이름이 **`index.html`** 인가 (오타·대문자 X)
- [ ] 경로는 **상대경로**만 — `assets/...` (맨 앞 `/` X · GitHub Pages에서 깨져요)
- [ ] **F12 → Console** 에 빨간 에러 0개
- [ ] **휴대폰(좁은 화면)** 에서도 보이나
- [ ] 폴더 이름·경로에 **한글·공백** 없나
- [ ] 소리를 넣었다면 경로가 `assets/audio/sfx/...` 맞나
- [ ] GitHub **Settings → Pages → Branch: main /(root)** 저장했나

## 배포 (GitHub Pages)
> 수업 저장소는 함께 쓰는 것이라 바로 못 올려요. **내 폴더로 복사** → 내 저장소로.
1. 이 `game` 폴더를 **바탕화면의 새 폴더**(영어 이름)로 복사 → VS Code로 **Open Folder**
2. **Source Control → Commit → Publish to GitHub → public**
3. GitHub **Settings → Pages → Branch: main /(root) → Save**
4. 1~2분 뒤 `https://내아이디.github.io/내저장소/` 에서 전 세계가 플레이!

> 💡 Phaser 3 게임 엔진을 쓰려면 `index.html` 상단의 주석 처리된 `<script ...phaser...>` 줄의 주석을 풀어요.
